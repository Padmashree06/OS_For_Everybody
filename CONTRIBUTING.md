# Contributing to OS_Algo_Simulator 

Thank you for your interest in contributing to **OS_Algo_Simulator**!  
This project is built to help students understand Operating System algorithms through simulations, and all contributions are welcome.

---

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button on the top-right corner of this repository to create your own copy.

---

### 2. Clone Your Fork
```bash
git clone https://github.com/<your-username>/OS_Algo_Simulator.git
cd OS_Algo_Simulator
```

---

### 3. Create a New Branch
Always create a new branch for your changes.
```bash
git checkout -b feature-name
```

---

### 4. Make Your Changes
You can contribute by:
- Adding new Operating System algorithms
- Improving existing simulations
- Fixing bugs
- Enhancing UI/UX
- Improving documentation

Make sure your code is clean, readable, and follows the existing project structure.

---

## Submitting a Pull Request

1. Commit your changes:
```bash
git commit -m "Add: short description of your change"
```

2. Push your branch:
```bash
git push origin feature-name
```

3. Open a **Pull Request** to the `main` branch of this repository.

4. In your PR description, clearly mention:
- What the change does
- Why it is needed
- Screenshots (if UI-related)

---

##  Reporting Bugs

If you find a bug, please open an issue and include:
- A clear description of the bug
- Expected behavior
- Actual behavior
- Steps to reproduce
- Screenshots or logs (if applicable)

---

## Feature Requests

Feature requests are welcome!
When opening an issue, include:
- The problem you are trying to solve
- Your proposed solution
- Any alternatives you considered

---

## Code Guidelines

- Follow the existing coding style
- Use meaningful variable and function names
- Write clear and concise commit messages
- Comment complex logic where necessary

---

## Documentation

If your contribution changes functionality:
- Update the README or relevant documentation
- Add examples or comments if required
